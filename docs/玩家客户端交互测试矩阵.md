# 玩家客户端交互测试矩阵

## 目标与判定原则

本矩阵只从真实玩家客户端视角验收，不以服务端内部状态或无界面探针代替最终结论。每个多客户端用例至少记录房主、行动玩家、旁观玩家三个视角的截图或日志。

全局不变量：

- 房主是唯一牌局权威；客户端不能直接改写座位、筹码、底牌、公共牌或当前行动者。
- 未公开牌只在所属玩家客户端可见；摊牌或主动亮牌后才向其他客户端公开。
- 不轮到本地玩家、已弃牌、已全下、暂离或未参与本手时，操作面板必须隐藏。
- 玩家离开、断线、重连或中途加入不能重置正在进行的牌局。
- 同一动作只结算一次；同一底池份额只产生一次收池记录。
- 每一手结束后，所有临时亮牌、下注、行动提示和计时状态必须重置。

状态说明：

| 标记 | 含义 |
|---|---|
| `已验证` | 已通过可视玩家客户端操作验证 |
| `待执行` | 当前版本可执行，但尚未完整回归 |
| `预期失败` | 当前缺少必要实现，执行应形成缺陷单 |
| `待公网` | 需要手机、不同网络或公网服务环境 |

## 覆盖维度

测试组合至少覆盖：

| 维度 | 取值 |
|---|---|
| 客户端角色 | 房主、普通玩家、非当前行动玩家、当前行动玩家 |
| 牌局阶段 | 大厅、翻牌前、翻牌、转牌、河牌、亮牌窗口、结算动画、手间等待 |
| 玩家状态 | 正常、弃牌、全下、暂离、零筹码、断线、重连、中途加入 |
| 网络状态 | 正常、延迟、短暂断开、永久断开、房主断开 |
| 桌型 | 9 人桌、12 人桌、空座、满桌 |
| 操作 | 过牌、跟注、下注、加注、全下、弃牌、亮一张、亮两张、补码、离开 |
