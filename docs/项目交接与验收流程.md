# TexasHoldem-Godot 项目交接与验收流程

## 当前完成状态

已完成可在本机直接落地的部分：

- 扁平化 UI：主菜单、大厅、牌桌、下注面板、玩家 HUD、聊天面板统一为低层级面板与按钮
- 自适应布局：牌桌保留顶部状态栏、右侧工具栏、底部下注区，玩家座位按可用区域重新计算；已针对 1920x1080 和窄屏/手机尺寸做紧凑模式
- 可收起侧栏：聊天和调试工具放在右侧栏，侧栏可收起，避免遮挡牌桌与下注按钮
- 明确退出路径：大厅有“退出房间”，牌桌有“离开牌桌”，主菜单有“退出游戏”
- AI 调试入口：大厅和牌桌右侧工具栏都可以选择是否加入 AI 测试玩家及 AI 数量
- 局内反馈：当前行动玩家 HUD 高亮并弹动，中央牌桌提示“轮到你行动/等待某玩家”
- 胜负反馈：结算后显示中心结果横幅，赢家 HUD 高亮并显示赢取筹码
- 牌型反馈：开牌后玩家手牌下方显示高牌、对子、两对等中文牌型
- 操作保护：非本地玩家行动时下注面板变暗、按钮禁用，避免点到后方或误操作
- 下注修复：确认按钮会在无人下注时提交“下注”，已有下注时提交“加注”
- Tween 动画：公共牌发牌、翻牌、玩家弃牌盖牌、赢家弹动均使用 Godot Tween
- 真实素材：已接入素材目录中的牌面、牌背、筹码图标，以及发牌/弃牌/筹码音效
- 桌面筹码上限接口：大厅和牌桌调试栏可设置桌面筹码上限，超额筹码回收到玩家账户字段并显示在 HUD
- Godot C# 工程：`TexasHoldemGodot.csproj`
- Godot.NET.Sdk：`4.6.3`
- 核心扑克逻辑：牌、牌堆、牌型评估、下注轮、主池/边池
- 测试场景：`res://tests/all_tests.tscn`、`TestHandEvaluator.tscn`、`TestBettingRound.tscn`、`TestPotManager.tscn`
- AutoLoad：`GameManager`、`NetworkManager`、`PlayerData`、`AudioManager`、`SettingsManager`
- 网络骨架：Host 权威、ENet 房间创建/加入、RPC 动作提交、状态同步 DTO
- 二维码生成：ZXing.Net 0.16.9
- UI 场景：主菜单、大厅、牌桌、玩家 HUD、下注面板、卡牌显示、公共牌、聊天、设置
- 资源：真实 PNG 卡牌、卡背、筹码，牌桌/UI 面板 SVG，以及 OGG 音效

## 真实联机验收

需要两台电脑在同一 WiFi 或局域网：

1. A 电脑打开项目，点击“创建房间”。
2. 记录大厅显示的本机 IP 和房间号，或扫描二维码查看 JSON。
3. B 电脑打开项目，点击“加入房间”。
4. 输入 A 电脑 IP，端口使用默认 `7000`。
5. A 电脑点击“开始游戏”。
6. 验证 B 电脑同步进入牌桌。

二维码内容格式应类似：

```json
{"v":1,"ip":"192.168.1.100","port":7000,"room":"123456"}
```

## 3D 表现路线

当前版本仍是 2D UI + Tween 表现，未直接切换为 3D。建议下一阶段按以下顺序推进：

1. 新建 `Table3D` 场景，保留现有 `GameManager`、`BettingRound`、`HandEvaluator` 等逻辑不动。
2. 用 `MeshInstance3D` 或薄盒体生成 3D 卡牌，把当前 PNG 牌面作为 `StandardMaterial3D` 的 Albedo 贴图。
3. 用 `AnimationPlayer` 或 Tween 控制发牌路径、翻转、弃牌归堆，先做单机表现，再接多人同步。
4. 将现有 `GameTable` 的下注面板、聊天、调试侧栏保留为 2D Overlay，避免 3D 化后影响可操作性。
