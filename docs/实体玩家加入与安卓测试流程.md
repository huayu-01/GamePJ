# 实体玩家加入与安卓测试流程

当前客户端已移除局域网广播、私网段扫描、防火墙配置和私网邀请信息。实际牌局仍使用 Godot ENet，但正式联机目标改为所有玩家连接公网权威服务器，不再由某位玩家的手机或电脑充当可被外网访问的 Host。

## 当前状态

- 主菜单保留自动搜房 UI 和 `IRoomDirectoryProvider` 接口。
- 公网房间目录尚未接入时，自动搜房会显示明确提示。
- 手动公网地址仅作为开发调试入口，不作为正式玩家流程。
- 客户端仍会校验应用版本和网络协议版本。
- 客户端只接收自己的手牌、已发出的公共牌和已公开手牌。

公网服务的实施顺序、接口和验收条件见 [公网联机方案清单.md](公网联机方案清单.md)。

## 安卓测试 App

当前项目使用 Godot 4.6.3 .NET、.NET 9、OpenJDK 17 和 Android SDK 进行构建。Android 导出预设名为 `Android Test`，目标为 ARM64 竖屏设备。项目内置模拟器、系统镜像和虚拟设备数据已经移除，后续使用实体 Android 设备验收。

导出带版本号 APK：

```powershell
powershell -ExecutionPolicy Bypass -File tools/android/export_versioned_apk.ps1
```

导出后在实体手机中打开 `build/android/GamePJ-<版本号>.apk` 安装。需要命令行安装时，可使用保留的 `tools/android-sdk/platform-tools/adb.exe install -r <APK路径>`。

## 公网联机验收

1. 部署房间目录 API 和至少一个公网 Godot 权威服务器。
2. 两台设备分别使用 Wi-Fi 和移动网络登录。
3. 设备 A 创建房间，目录中出现房间号。
4. 设备 B 打开加入页面，自动搜索到房间并加入。
5. 服务端开始牌局，两端座位、行动、公共牌和底池一致。
6. 任一客户端切换网络或短暂断线后，可凭会话令牌恢复原座位。
7. 客户端无法获取其他玩家未公开手牌或未发出的公共牌。
8. 客户端版本或协议不兼容时，目录隐藏房间或给出升级提示。
