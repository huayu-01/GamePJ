# 实体玩家加入与安卓测试流程

这份文档用于让第二个真实玩家尽快加入测试桌。当前项目使用 Godot ENet 直连模型：Host 创建房间，其他玩家用 Host 的 IP 和 UDP 端口加入。

## 当前可用流程

### 同一 Wi-Fi / 局域网

1. Host 打开游戏，点击 `创建房间`。
2. Host 在房间页点击 `复制邀请信息`。
3. 把复制出来的邀请信息发给另一个玩家。内容类似：

```json
{"v":1,"ip":"192.168.1.100","port":7000,"room":"123456","max_players":9}
```

4. 加入玩家打开游戏，点击 `加入房间`。
5. 粘贴邀请信息，点击 `连接`。
6. Host 在房间页确认玩家列表，然后点击 `开始游戏`。

如果不想复制 JSON，也可以只填：

```text
192.168.1.100:7000
```

默认端口是 UDP `7000`。

### 不同网络

不同网络不能只填局域网 IP，例如 `192.168.x.x` 或 `10.x.x.x`，这些地址只在 Host 自己的局域网内有效。

可行方式：

1. Host 使用有公网 IP 的网络，或通过路由器做端口转发。
2. 路由器把 UDP `7000` 转发到 Host 电脑的局域网 IP。
3. 加入玩家填写 Host 的公网 IP：

```text
公网IP:7000
```

如果 Host 网络是运营商 CGNAT，或者公司/校园网禁止入站 UDP，直连通常不可用。后续应接入中继服务器或房间服务器。

## 安卓玩家如何拿到 App

测试 APK 发布在 GitHub Release：

```text
https://github.com/huayu-01/GamePJ/releases/tag/v0.1.0-test
```

下载 `GamePJ-test.apk` 后安装即可。当前包为 ARM64 调试包，支持 Android 7.0（API 24）及以上版本，包名为 `com.huayu.gamepj`。

### 开发测试安装

1. 在手机设置中允许浏览器或文件管理器安装未知来源应用。
2. 下载并打开 `GamePJ-test.apk`。
3. 已安装旧测试版时可直接覆盖安装；签名不一致时先卸载旧版。
4. 首次进入后，与 Host 在同一 Wi-Fi 下粘贴邀请信息进行联机测试。

连接了 USB 调试设备时，也可以执行：

```powershell
adb install -r build/android/GamePJ-test.apk
```

### 本机构建环境

当前项目已经提交 `export_presets.cfg`，Android 预设名为 `Android Test`。已验证的构建环境：

- Godot 4.6.3 .NET/Mono
- .NET SDK 9.0.315
- Microsoft OpenJDK 17
- Android SDK Platform 36
- Android Build Tools 36.1.0
- Godot 4.6.3 Mono export templates

导出命令：

```powershell
Godot_v4.6.3-stable_mono_win64_console.exe --headless --path . --export-debug "Android Test" "build/android/GamePJ-test.apk"
```

### 推荐分发方式

短期测试：

- 把 debug APK 发到群文件、网盘或 GitHub Release。
- 每次发包附带版本号和提交号。

后续内测：

- 使用 GitHub Actions 自动打包 APK。
- APK 上传到 GitHub Release。
- 玩家只需要下载最新 Release 的 APK。

正式发布：

- 使用 release keystore 签名。
- 走应用商店或自建下载页。

## 当前项目已支持/已预留

- 主菜单加入页支持粘贴邀请 JSON。
- 主菜单加入页支持 `IP:端口`。
- 连接失败不会直接跳房间，会在加入页显示失败原因。
- Host 开始游戏后，客户端会自动进入牌桌。
- Host 会向每个客户端单独发送该玩家自己的两张手牌。
- 公共桌面状态通过 Host 同步，客户端不会收到其他玩家未公开手牌。

## 仍需后续完善

- 公网中继：当前是直连 ENet，没有 relay/大厅服务器。
- 掉线重连：当前断线后需要重新加入。
- 玩家名/头像：当前注册玩家名，头像仍未接入。
- Android 自动构建：已有本地导出预设，仍需补 GitHub Actions 自动打包脚本。

## 测试清单

同 Wi-Fi 测试：

1. 电脑 A 创建房间并复制邀请信息。
2. 电脑 B 或安卓设备粘贴邀请信息。
3. Host 房间页能看到新玩家。
4. Host 点击开始游戏。
5. 客户端自动进入牌桌。
6. 客户端只能看到自己的手牌、公共牌和已公开信息。
7. 客户端操作按钮只在轮到自己时可用。

跨网络测试：

1. Host 路由器转发 UDP `7000` 到 Host 电脑。
2. 使用手机移动网络加入公网 IP。
3. 如果失败，先确认端口转发、Windows 防火墙和运营商是否提供公网 IP。
